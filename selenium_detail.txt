Selenium - 
	инструмент для автоматизированого управления браузерами 
	||
	Драйвер для браузера (ПО для управления браузером(запустить, открыть страницу,найти ссылку, кликнуть,закрыть)). Умеет только работать с браузером.

Selenium WebDriver -
	По назначению Selenium WebDriver представляет собой драйвер браузера, то есть программную библиотеку, которая позволяет разрабатывать программы, управляющие поведением браузера.
	
Принципы работы -
    (говоря про js) 
	Схема 1 - все внутри (расширения для браузера, управление через js)
	Схема 2 - внедрения "сзади" т.е. прокси который будет между сервером и нашим браузером.
	Схема 3 - внедрение "спереди"
	
	Интерфейсы для удаленного управления браузерами
		- Remote Debug для Chrome
		- Marionette для FireFox
		- Com API для IE
	У каждого браузера есть вспомогательный исполняемый файл, который наружу предоставляет универсальный интерфейс селениум.
		- сhromedriver для Chrome
		- geckodriver для FireFox
		- IEDriverServer для IE
	Таким образом цепочка взаимодействия
		SE -> сhromedriver -> Remote Debug(Chrome)
		SE -> geckodriver -> Remote Debug(FireFox)
		SE -> IEDriverServer -> Com API(IE)

Что описывает стандарт W3C Webdriver -
	- Набор команд 
	- Алгоритмы (пределения скрытых элементов, текста на странице)
	- Сетевой протокол
	- Механизмы расширения протокола (Для мобильного тестирования)
	
Кокуренты и партнеры - 
	Альтернативы для драйвера
		- Sahi (индийская разработка) использует схему "прокси"
		- Watir - classic(надстройка над селениумом)
		- Проприетарные инструменты (Visual Studio)
	
	Альтернативные API	(реализуют стандарт W3C Webdriver)
		- реализация клиентских библиотек
		- Fluent(fluent-selenium, WD.js)(последовательное вытягивание элементов страницы)
		- Типизированные (Watir, HtmlElements)
		- Процедурные (Selenide, Webium)
